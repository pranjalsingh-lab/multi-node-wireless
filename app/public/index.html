Multi-Node Simulation

Three emulated Nordic nRF52840 nodes · virtual Bluetooth LE network · live firmware sandbox
Idle
How this works read me ▸

This lab boots three virtual Nordic nRF52840 boards and runs real firmware on emulated hardware - there is no physical device anywhere. Two of the boards talk to each other over a virtual Bluetooth LE radio; the third reads a simulated accelerometer over SPI. Each card shows that node's live serial console, exactly what the firmware prints.

The three nodes

  • Gateway - BLE central. Scans the air, connects to the band, and collects its readings.
  • Heart-Rate Band - BLE peripheral. Advertises a heart-rate service and streams beats per minute.
  • Motion Sensor Node - reads an ADXL372 accelerometer over SPI. No radio.

When you press Start

  1. All three boards power on and boot their firmware.
  2. The band starts advertising; the gateway starts scanning.
  3. They find each other and connect over Bluetooth LE.
  4. The gateway subscribes and heart-rate notifications begin flowing.
  5. The motion node reads its sensor on a loop.

Expect this to settle within a few seconds.

Reading the consoles

Each dark box is the raw serial output of one board. The green › lines are plain-language milestones we add on top (“connected”, “streaming…”) so you can follow along without parsing every line. The badge top-right of each card tracks that node's current state.

Things to try

  • Drag the motion node's X / Y / Z sliders - the reading the firmware sees changes live.
  • Hit View code to read the exact firmware source and how it communicates.
  • Compile your own image and Upload .elf to run it; Default reverts.
Everything runs on virtual hardware in your browser session - no boards, no wires, no flashing.
